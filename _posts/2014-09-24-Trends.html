the one titled: building my first twitter app.

I pulled a one man hackathon this past Sunday night to build my first twitter trends app. The app's user interface is very simple-- a list of Nairobi's trending Twitter topics.

I built the app on the Python Google App Engine platform and included the Python Twitter Tools package (installable by running the command 'pip install twitter' on a mac terminal) because of the ease with which it allowed me to pull data from Twitter's stream API. Here's a helpful tutorial Mining the Social Web' 2nd Edition detailing the steps needed to use the said package to access Twitter's data.

Since Twitter implements OAuth 1.0A as its standard authentication mechanism, I needed to login to Twitter and create a twitter application. After contructing MapsAndTrends, my first Twitter app, I noted its primary identifiers necessary for an OAuth 1.0A workflow: consumer key, consumer secret, access token, and access token secret. Once authentication was guaranteed, I wrote a simple python script to pull trending data and display it. I would later integrate this script into a Google App Engine instance, to keep it alive in the cloud.

The script, twitterStream.py, performed Oauth authentication, loaded the entire world, Kenya and the USA's WOEIDs-- Where On Earth Identifiers--, used them to pull trending data and printed this data out to standard output.

The Google App Engine framework basically hosted twitterStream.py within a webapp2 application that handled all the code I'd need for the WSGI (Web Server Gateway Interface). I then extended twitterStream.py to write the trending data out to the app's home page.

The next step for this project will be to visualise the data and locate it on a map. Here I'll select a data visualisation library to leverage. As concerns locating the data, I've already decided on Google Map's javascript Web API as my mapping library of choice because it's properly documented and easy to use. This brings me to my first concern of the project: all web data visualisation APIs I've come across thus far are in javascript as well.

This has pushed me to look at a couple of Node.js modules for an alternative platform on which to build my web app. I've came across phonegap and ionic. These two frameworks enable developers to build web apps that can be compiled into downloadable mobile phone apps on iOS, Android®, Windows© Phone, Blackberry© and webOS platforms whereby they'll run natively.

This would make the project more accessible to mobile phone users while easing the work of integrating javascript libraries into it.
